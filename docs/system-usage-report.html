一、系统概览

本系统是一个完整的 B2C 多商家电商平台，支持 买家购物、商家入驻经营、管理员后台监管 三大角色体系。 涵盖商品浏览与搜索、多商家购物车拆单、订单全流程管理、售后服务、物流追踪、人工客服、AI 智能客服等核心功能。

买家 (user)

浏览商品、加购物车、下单支付、收藏、售后申请、在线客服

商家 (merchant)

商品管理、订单发货、售后处理、店铺设置、AI 知识库、客服

管理员 (admin)

用户管理、商家审核、商品/分类管理、订单监管、售后监管、数据看板

二、技术架构

层级 技术选型 说明
前端 Vue 3 + TypeScript + Pinia + Tailwind CSS v4 + Vite Composition API，响应式 SPA
后端 FastAPI + Pydantic v2 + Psycopg3 异步框架，自动生成 OpenAPI 文档
数据库 PostgreSQL 16 (pgvector) 支持向量检索，AI 客服语义搜索

三、部署与启动

3.1 Docker Compose 一键启动

# 进入项目目录
cd D:\AI_Projects\test01

# 构建并启动所有服务（首次）
docker compose up -d --build

# 后续启动（已有镜像）
docker compose up -d

# 查看服务状态
docker compose ps

# 查看后端日志
docker compose logs -f shop-service

# 停止所有服务
docker compose down

3.2 前端开发服务器（热更新）

# 进入前端目录
cd shop-frontend

# 安装依赖（首次）
npm install

# 启动 Vite 开发服务器
npx vite --port 5173

四、买家端使用指南

4.1 注册与登录

  1. 1 访问 /login 页面，点击「注册账号」
  2. 2 填写邮箱、密码（需字母+数字组合）、昵称、密保问题及答案
  3. 3 注册成功后自动跳转登录页，输入邮箱密码登录
  4. 4 系统根据角色自动跳转：买家 → 首页 /

4.2 商品浏览与搜索

访问 /products 商品列表页，支持以下筛选方式：

关键词搜索

输入商品名称模糊搜索

分类筛选

按树形分类逐级浏览

价格区间

设置最低/最高价格

排序方式

价格升/降序、销量、最新

仅看有货

过滤库存为 0 的商品

热门商品

首页展示热销商品

点击商品进入详情页 /products/:id，可查看图片、价格、库存、描述，并进行加购、收藏、咨询客服。

4.3 购物车与下单

加入购物车

在商品详情页选择数量后点击「加入购物车」，系统自动按商家分组展示。

购物车页面 (/cart)

  • 商品按商家分组展示，每个商家有独立标题栏
  • 支持按商家全选，也可单独勾选商品
  • 底部显示已选商品数和总价
  • 结算时如有多个商家的商品，会提示「将拆分为 N 个订单」

下单结算流程

  1. 点击「结算」→ 选择收货地址（支持地址簿管理）
  2. 确认订单信息 → 提交订单
  3. 系统自动按 merchant_id 拆分独立订单（每个商家一个订单）
  4. 跳转订单列表，分别支付各订单
多商家拆单说明：不同商家的商品会自动拆分为独立订单，每个订单有独立的物流、支付、售后流程。这是系统的核心设计，确保各商家独立运营。

4.4 订单管理

访问 /orders 查看全部订单，支持按状态筛选。

订单状态 含义 买家可操作
pending待支付支付 / 取消
paid已支付（待发货）等待商家发货
shipped已发货查看物流 / 确认收货
delivered已收货申请售后
cancelled已取消—

点击「查看物流」可进入物流详情页 /orders/:id/logistics，查看配送进度。

4.5 其他功能

收藏夹 /favorites

在商品详情页点击心形图标收藏，收藏夹页面可管理已收藏商品，支持取消收藏。

个人中心 /profile

修改昵称、头像、收货地址、密码。支持通过密保问题找回密码。

地址簿

管理多个收货地址（省/市/区/详细地址），可设置默认地址。下单时快速选择。

在线客服

商品详情页可发起客服会话（自动路由到对应商家），无商品时路由到系统管理员。支持 AI 客服 + 人工客服。

五、商家端使用指南

5.1 商家入驻流程

  1. 1 在登录页点击「商家入驻」进入 /merchant-register
  2. 2 填写邮箱、密码、昵称、店铺名称、经营类目
  3. 3 上传营业执照和身份证（可选，图片转 Base64 存储）
  4. 4 设置密保问题和答案
  5. 5 提交后进入「等待审核」页面 /merchant-pending，页面自动轮询审核状态
  6. 6 管理员审核通过后，可登录商家后台 /merchant
审核状态： pending（待审核）→ approved（通过，可登录） / rejected（拒绝，显示拒绝原因）

5.2 商家仪表盘

访问 /merchant 进入商家仪表盘：

今日订单

当日新增订单数

在售商品

status=on_sale 的商品数

待发货

status=paid 的订单数

本月销售额

当月已支付订单总金额

同时展示销售额走势图（SVG 面积图，最近 7 天）、店铺信息、最近订单列表。

5.3 商品管理

访问 /merchant/products 管理商品：

  • ● 新增商品：填写名称、价格、库存、分类、图片 URL、描述、折扣价
  • ● 编辑商品：修改商品信息
  • ● 上下架：切换 on_sale / off_sale 状态
  • ● 删除商品：永久删除（关联订单不受影响）
注意：新建商品默认为 pending 状态，需管理员审核通过后才会上架（on_sale）。

5.4 订单管理

访问 /merchant/orders 管理订单：

  • ● 查看属于自己的订单（按 merchant_id 过滤）
  • ● 对 paid 状态订单进行发货，填写物流信息（快递公司 + 运单号）
  • ● 发货后订单状态变为 shipped，等待买家确认收货

5.5 售后处理

访问 /merchant/after-sales 处理买家售后申请：

操作 说明
approve同意售后申请（退款/换货）
reject拒绝售后申请，需填写拒绝原因
complete完成售后（退款已到账 / 换货已完成）
resend换货场景：商家重新发货给买家
confirm-received确认收到买家退回的商品

5.6 店铺设置

访问 /merchant/profile 修改店铺信息：店铺名称、店铺描述、店铺 Logo。买家端会展示这些信息。

6.2 用户管理

访问 /admin/users 管理所有用户：

  • ● 查看所有用户列表（买家/商家/管理员）
  • ● 商家审核：批准 / 拒绝商家入驻申请（可填写拒绝原因）
  • ● 重置密码：为用户重置密码
  • ● 删除用户：删除违规用户

6.3 商品审核与管理

访问 /admin/products：

  • ● 查看所有商品（全平台）
  • ● 商品审核：批准 / 拒绝商家新建的商品（pending → on_sale / rejected）
  • ● 强制下架：违规商品可强制改为 off_sale
  • ● 删除商品
管理员在商城页面为「预览模式」：管理员可以浏览商品，但隐藏加购/购买/收藏/结算按钮。

6.4 分类管理

访问 /admin/categories：

  • ● 支持树形分类结构（parent_id 自关联）
  • ● 新增 / 编辑 / 删除分类
  • ● 设置父级分类实现多级嵌套

6.5 订单与售后监管

管理员可查看全平台订单 /admin/orders 和售后记录 /admin/after-sales， 但为「监管模式」——只查看不操作。订单发货、售后处理由商家完成，管理员仅监督买卖双方行为。

6.6 邮箱换绑审核

访问 /admin/email-changes 审核用户提交的邮箱更换申请，可批准或拒绝。

八、人工客服系统

8.1 会话路由规则

  • 有 product_id → 路由到对应商家（商品归属的 merchant）
  • 无 product_id → 路由到系统管理员（系统级咨询）

8.2 各角色客服入口

角色 入口 功能
买家商品详情页右下角悬浮按钮发起会话、发送消息、查看历史
商家/merchant/chat查看会话列表、回复消息、关闭会话
管理员/admin/chat处理系统级会话、回复消息

8.3 技术特性

  • ● 增量轮询：前端通过 after_id 参数只拉取新消息，减少带宽
  • ● 归属权验证：各角色只能访问自己的会话，越权返回 403
  • ● 乐观更新：前端发送消息时立即显示（占位 ID），服务端返回后替换
  • ● 已读标记：支持标记消息已读，显示未读计数
  • ● 会话关闭：商家/管理员可关闭已完结的会话

九、AI 智能客服

系统内置 AI 智能客服，基于通义千问大模型 + RAG（检索增强生成），支持知识库问答。

9.1 知识库管理

管理员知识库

入口：/admin/ai-chat

上传系统级知识文件（FAQ、政策等），对所有商家和买家生效。

商家知识库

入口：/merchant/ai-chat

上传店铺专属知识（商品说明、退换政策等），仅对该商家的买家生效。支持按商品维度关联。

9.2 工作原理

买家提问
   │
   ▼
┌─────────────────────────┐
│ 1. 向量检索知识库        │  pgvector 语义搜索 top_k=3
│    (kb_chunks 表)        │
├─────────────────────────┤
│ 2. 组装 Prompt           │  系统提示 + 检索到的知识片段 + 用户问题
├─────────────────────────┤
│ 3. 调用通义千问 API      │  生成回答
├─────────────────────────┤
│ 4. 返回回答 + 引用来源   │  买家端展示
└─────────────────────────┘

买家在商品详情页的 AI 客服对话框提问，系统自动检索商家知识库 + 系统知识库，生成准确回答。
