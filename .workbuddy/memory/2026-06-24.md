## 前端页面恢复 + 数据库重建

### 问题
前端页面拒绝访问（5173 端口无进程），原因是：
1. Vite 开发服务器进程因会话断开而退出
2. Docker 容器重建导致 shop-service 启动时连不上 postgres（旧进程残留），数据库回退到 init.sql 初始状态
3. 数据全部丢失 + products 表缺少 approval_status/merchant_id/discount/sales 列

### 修复步骤
1. 用 `node node_modules/vite/bin/vite.js --host 0.0.0.0` 重新启动前端（npx 因 PowerShell 执行策略失败，直接 node 调用 vite.js 成功）
2. 重启 shop-service 容器
3. 执行 add_merchant_role.sql 和 add_product_discount_sales.sql migration 补齐表结构
4. 手动插入 10 条商品数据 + 9 条分类数据（init.sql 只在首次初始化执行，pgdata volume 已存在导致跳过）
5. 验证前后端 API 均正常返回 10 条商品

### 关键发现
- **pgdata volume 持久化问题**：容器重建后 init.sql 不会重新执行，需要手动插入数据
- **Windows Git Bash 下 npx 不可用**：需用 `node node_modules/vite/bin/vite.js` 启动 Vite
- 建议后续将 migration 脚本整合到 init.sql 中，或添加一个 docker-entrypoint 后置脚本自动执行 migration
