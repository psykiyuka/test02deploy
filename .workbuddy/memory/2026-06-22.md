# 2026-06-22 工作记录

## 电商系统项目 - 修复前端商品不显示问题

### 问题诊断
- 前端页面访问 http://localhost:5173 不显示商品
- 根因：数据库 `shop.products` 表缺少后端代码依赖的列（`approval_status`、`merchant_id`、`discount`、`sales`）
- `init.sql` 建表脚本是旧版本，没有这些新加的列
- 后端 SQL 查询使用了 `p.approval_status = 'approved'` 过滤，导致查询失败

### 修复步骤
1. 执行 `shop-service/migrations/add_merchant_role.sql` — 添加 `merchant_id`、`approval_status`、`shop_description`、`shop_logo` 等列
2. 执行 `shop-service/migrations/add_product_discount_sales.sql` — 添加 `discount`、`sales` 列并生成模拟销量
3. 启动前端开发服务器 `npm run dev`（端口 5173）

### 验证结果
- API `http://localhost:8004/api/shop/c-endpoint/products` 返回 10 条商品
- 前端代理 `/api/shop` → `http://localhost:8004` 正常
- 前端页面 http://localhost:5173 应能正常显示商品

### 重要提示
- Windows 上需用 `docker-compose`（带连字符）而非 `docker compose`
- 数据库表都在 `shop` schema 下，需用 `shop.products` 查询
- 未来若重建容器（`docker-compose down -v`），需重新运行 migration 脚本
