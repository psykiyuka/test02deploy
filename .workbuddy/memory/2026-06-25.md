
## 修复地图选址逆地理编码不生效（2026-06-25 16:20）

### 问题
点击地图后底部不显示识别结果，仍显示"点击地图选择收货位置"。

### 根因
高德 JS API 2.0 的 Web 服务（如 Geocoder）需要配合 `_AMapSecurityConfig` 安全密钥。之前只通过 script src 的 `securityJsCode` 参数加载地图，但逆地理编码请求需要单独的 `_AMapSecurityConfig` 配置，否则会被高德服务端拦截返回错误。

### 修复
1. 在 `loadAMapScript()` 开头设置 `window._AMapSecurityConfig = { securityJsCode: '...' }`
2. 即使 AMap 已加载也确保安全密钥被设置
3. 逆地理编码回调中加入 console.log 调试输出，方便排查

## 输入框美化（2026-06-25 01:30）

### 问题
用户反馈：各界面输入框的提示词在输入数据后有遮挡。

### 根因
`Login.vue` 使用浮动标签（float label）模式，但 label 的 CSS 类只处理了 `peer-focus:` 状态，
没有处理 `peer-not-placeholder-shown:` 状态，导致输入内容后输入框失焦，label 又盖回去遮挡文字。

### 修复内容
1. **Login.vue 浮动标签修复**：给所有 label 加上完整的 Tailwind 类：
   - 默认（空值）：`top-1/2 -translate-y-1/2 text-sm text-gray-400`（居中）
   - 有内容（`peer-not-placeholder-shown`）：`top-1.5 text-xs text-indigo-600 font-medium`（上浮）
   - focus（`peer-focus`）：同上
   - 加了 `placeholder-transparent` 隐藏原生 placeholder

2. **Login.vue input 统一美化**：所有 input 加上 `pt-5 pb-2 text-sm text-gray-900 hover:border-gray-300 transition-all duration-200`

3. **Profile.vue 密码框增强**：
   - 加上密码显示/隐藏按钮（Eye/EyeOff 组件）
   - 加上 `showOldPassword`/`showNewPassword`/`showConfirmPassword` 状态变量
   - 引入 `Eye, EyeOff` 图标

4. **merchant/Profile.vue 统一**：
   - input class 统一为 `rounded-xl px-4 py-3 pr-12`（之前是 `rounded-lg pr-10`）
   - 密码按钮图标从 raw svg 改为 `Eye`/`EyeOff` 组件
   - 按钮样式统一为 `right-4 p-0.5 rounded-md hover:bg-gray-100`

### 统一输入框样式规范（后续参考）
- 普通 input：`px-4 py-3 rounded-xl border border-gray-200 text-sm placeholder:text-gray-400`
- focus：`focus:ring-2 focus:ring-indigo-500 focus:border-transparent`
- hover：`hover:border-gray-300`
- 密码 input：加 `pr-12` 给按钮留空间
- 浮动标签：input 加 `placeholder=" " placeholder-transparent`，label 用 `peer-placeholder-shown:` + `peer-not-placeholder-shown:` 控制位置
