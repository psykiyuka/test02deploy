## 修复：支付成功后不显示结果 + 已付款订单仍显示"立即支付"

**根本原因**：支付宝沙箱异步通知（notify）无法到达 localhost，导致用户在支付宝端完成支付后，本地订单状态始终停留在 pending，前端轮询也检测不到 paid 状态变化。

**修改内容**：

1. **后端新增主动查询接口** `GET /c-endpoint/payment/alipay/query/{order_id}`：
   - 调用 `alipay.api_alipay_trade_query()` 主动查询支付宝端交易状态
   - 如果支付宝端显示 TRADE_SUCCESS/TRADE_FINISHED，自动调用 `pay_order` 更新本地订单状态为 paid
   - 如果本地已是 paid，直接返回成功
   - 查询失败不影响正常流程，返回当前本地状态

2. **前端 Payment.vue 轮询逻辑升级**：
   - `checkPayStatus()` 函数：先查本地订单状态，如果仍是 pending，再调用 `/alipay/query/` 主动查支付宝端
   - 扫码支付和网页支付轮询都使用新的 `checkPayStatus()`
   - 从支付宝回调回来时（`route.query.from === 'alipay'`），如果本地不是 paid，也主动查询

3. **Orders.vue 路由变化刷新**：
   - 添加 `watch(route.path)` 监听，从支付页返回订单页时自动 `fetchOrders()` 刷新数据
   - 确保支付完成后回到订单列表能看到正确的按钮状态（已付款显示"申请退款"而非"立即支付"）

## 修复：支付宝密钥格式

**问题**：用户配置 .env 时从支付宝沙箱复制了纯 base64 的私钥和公钥，没有 `-----BEGIN/END RSA PRIVATE KEY-----` 和 `-----BEGIN/END PUBLIC KEY-----` 标记，支付宝 SDK 抛 `RSA key format is not supported`。

**修改**：`payment/router.py` 中 `_get_alipay_client()` 新增 `_format_private_key()` 和 `_format_public_key()` 辅助函数，自动检测并补全密钥头尾标记（已包含则不处理）。容器已重启，代码已生效。

**说明**：用户需保证 .env 中的密钥是支付宝沙箱提供的真实密钥；支付宝公钥应为沙箱平台给出的 `MIGf...` 形式，应用私钥为工具生成的 `MIIE...` 形式。当前自动补全使用 PKCS#1 RSA 私钥标记和 X.509 PUBLIC KEY 标记，兼容沙箱默认配置。

## 项目代码阅读 - 电商平台 test02deploy

完整阅读了 `E:/hfinal_work/test02deploy` 项目代码，梳理了整体架构和功能布局，为后续问题修复做铺垫。

### 技术栈
- **前端**: Vue 3 + TypeScript + Pinia + Tailwind CSS v4 + Vite（端口 5173）
- **后端**: FastAPI + Python + PostgreSQL（pgvector/pg16）+ Redis
- **部署**: Docker Compose（Nginx 端口 83，后端映射 8004:8001）

### 后端结构（shop-service）
- `main.py`：FastAPI 入口，统一路由前缀 `/api/shop`
- `apps/c_endpoint/`：买家（Consumer）端 API，含 product/user/cart/order/logistics/after_sale/category/favorite/address
- `apps/b_endpoint/`：商家后台 API，含 category/product/order/after_sale/dashboard/user
- `apps/m_endpoint/`：商家注册/审核 API
- `apps/admin_endpoint/`：管理员端 AI 聊天等
- `apps/internal/`：内部接口（Nginx 层拦截，返回 403）
- `domain/`：业务逻辑层（order/product/user/cart/after_sale/logistics/category/ai_chat 等）
- `infrastructure/`：数据库连接池（psycopg2 ThreadedConnectionPool）、Redis 缓存、调度器（APScheduler）
- `common/`：日志、异常类（ValidationError/BusinessError/NotFoundError 等）
- 定时任务：每 5 分钟扫描并自动取消超时 30 分钟的 pending 订单

### 前端结构（shop-frontend）
- 路由 router.ts：三大布局（DefaultLayout/AdminLayout/MerchantLayout）
- 买家页面：Home/ProductList/ProductDetail/Cart/Orders/Logistics/AfterSale/Favorites/Profile/Login
- 管理员页面（/admin）：Dashboard/Categories/Products/Orders/AfterSales/Users/EmailChanges/AIChatSettings/Chat
- 商家页面（/merchant）：Dashboard/Products/Orders/AfterSales/Profile/AIChatSettings/Chat
- Stores：auth（登录/注销/token 管理）、cart、category
- API 工具：`src/utils/api.ts`，封装 fetch，base 取自 `VITE_API_BASE` 环境变量或默认 `/api/shop`

### 核心业务流程
1. 用户注册/登录 → JWT token 存 localStorage
2. 浏览商品（分类/搜索/热门） → 加购物车
3. 结算：选地址 → 按商家分组创建多个订单 → 模拟支付
4. 订单状态：pending → paid → shipped → delivered/cancelled
5. 物流：支付后自动生成物流记录，商家可更新物流状态
6. 售后：申请退款/换货 → 商家审核 → 买家退货 → 商家确认收货 → 完成
7. 商家入驻：注册申请 → 管理员审核（pending/approved/rejected）
8. 超时订单：APScheduler 每 5 分钟检查，30 分钟未支付自动取消并回滚库存

### 权限控制
- 路由守卫：admin/merchant 需对应 role；merchant 需 merchant_status 为 approved
- API 认证：Bearer JWT Token，401 时前端自动跳转 /login
- Nginx 层：/api/shop/internal/ 路径直接返回 403

### 数据库
- PostgreSQL schema: `shop`，pgvector 扩展
- 连接池：psycopg2 ThreadedConnectionPool (5~50)
- Redis 缓存：商品详情、热门商品列表

### 注意事项
- 图片/头像/营业执照以 base64 data URL 存入数据库
- 邮箱变更需管理员审核（EmailChanges）
- 商品上架需管理员审核（approval_status）
- CORS 设置为 allow_origins=["*"]，allow_credentials=False
